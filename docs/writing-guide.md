# Writing Guide

This repository is a technical blog, not a notes dump. Each post should be clear, useful, and easy to scan.

## Goal

Use the blog to document a real transition from legacy DBA work to platform engineering in a way that other engineers can follow.

## Writing Principles

- write in plain English
- keep paragraphs short
- prefer practical examples over abstract claims
- explain why something matters, not only what it is
- avoid hype, filler, and vague statements
- link to the source when a claim depends on external documentation

## Standard Post Structure

1. Start with a short opening paragraph that explains the point of the post.
2. Add 2 to 4 clear sections with descriptive headings.
3. End with a practical takeaway, next step, or lesson learned.

## Front Matter Rules

- use a direct and specific title
- keep categories limited to 1 or 2 items
- keep tags limited to 3 to 5 items
- use lowercase kebab-case tags
- keep dates in UTC when publishing manually

## Category Direction

Recommended categories for this site:

- Career
- Platform Engineering
- Database
- PostgreSQL
- Infrastructure
- Automation
- Web Development

## Quality Checklist

Before publishing, check the following:

- the title is specific
- the first paragraph explains the value of the post
- headings are short and useful
- commands and code examples are correct
- categories and tags are consistent with earlier posts
- the post says something concrete, not only motivational
